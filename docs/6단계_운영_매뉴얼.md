# 🚀 6단계 운영 매뉴얼

## 1. 초기 설치 및 실행

### 1-1. 최초 실행 (Docker)
```bash
cd /root/my_prog/asset_manager
docker compose up --build -d
```

### 1-2. 이후 실행
```bash
docker compose up -d
```

### 1-3. 종료
```bash
docker compose down
```

### 1-4. 접속
- 웹: `http://localhost:8090`
- SSH: `ssh -p 1040 root@localhost`

---

## 2. 개발 환경 실행

### 백엔드 개발 서버
```bash
cd /root/my_prog/asset_manager

# 가상환경 설정 (최초 1회)
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

# 실행
uvicorn backend.main:app --host 0.0.0.0 --port 8090 --reload
```

### 프론트엔드 개발 서버
```bash
cd /root/my_prog/asset_manager/frontend
npm install  # 최초 1회

npm run dev  # http://localhost:5173 에서 개발
```
> 개발 시 프론트(`5173`)는 백엔드(`8090`)로 프록시됨 (`vite.config.ts` 설정)

### 프론트엔드 빌드 및 배포 (컨테이너 내)
```bash
docker exec my-asset-manager bash -c "cd /app && bash start_server.sh"
```

---

## 3. DB 관리

### DB 파일 위치
```
/root/my_prog/asset_manager/data/assets.db
```

### DB 백업
```bash
cp data/assets.db data/assets_backup_$(date +%Y%m%d).db
```

### 컨테이너 내 DB 접근
```bash
docker exec -it my-asset-manager sqlite3 /app/data/assets.db
```

### 기존 Streamlit DB 이관
```bash
# 기존 DB가 backup_asset_manager/data/assets.db 에 있음
# 신규 프로젝트의 스키마는 동일하므로 그대로 복사 가능
cp /root/my_prog/backup_asset_manager/data/assets.db \
   /root/my_prog/asset_manager/data/assets.db
```

---

## 4. 주가 자동 업데이트

### 수동 업데이트 (UI)
- 주식 페이지 → "📈 주가 업데이트" 버튼 클릭

### API 호출
```bash
curl -X POST http://localhost:8090/api/stocks/update
```

### 자동 스케줄 (APScheduler)
- 평일 장 마감 후 자동 실행 (한국 기준 16:00, 미국 기준 22:30 이후)
- `backend/main.py`의 lifespan에서 스케줄 등록

## 5. 로그 확인

### 서버 로그
```bash
# 실시간 모니터링
tail -f /root/my_prog/asset_manager/logs/server.log

# 컨테이너 내
docker exec my-asset-manager tail -f /app/logs/server.log

# docker compose 로그
docker compose logs -f
```

## 6. 문제 해결 (Troubleshooting)

### 포트 충돌
```bash
lsof -i :8090
kill -9 <PID>
```

### 컨테이너 재시작
```bash
docker compose restart
```

### 프론트엔드 빌드 오류
```bash
cd frontend
npm install
npm run build
```

### DB 초기화 (주의: 모든 데이터 삭제)
```bash
rm data/assets.db
# 서버 재시작 시 자동 재생성
```

## 8. 데이터 입력 가이드

### 날짜 형식
- 모든 날짜: `YYYY-MM-DD` (예: `2024-01-15`)

### 금액 단위
- 모든 금액: 원(KRW) 기준
- 해외 주식: 단가는 외화(USD/JPY), 평가액은 자동으로 KRW 환산

### 자산 유형별 이력 기록 주기 권장
| 유형 | 권장 주기 |
|------|---------|
| 주식 | 자동 업데이트 (매일) |
| 부동산 | 월 1회 (시세 변동 시) |
| 연금 | 반기 1회 |
| 예적금 | 월 1회 |
| 실물자산 | 분기 1회 |
