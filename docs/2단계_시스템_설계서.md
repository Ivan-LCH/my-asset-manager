## 3. 데이터베이스 설계 (SQLite Schema)

### 3.1 assets (공통 자산 테이블)
```sql
CREATE TABLE IF NOT EXISTS assets (
    id                TEXT PRIMARY KEY,         -- UUID
    type              TEXT NOT NULL,            -- REAL_ESTATE|STOCK|PENSION|SAVINGS|PHYSICAL|ETC
    name              TEXT NOT NULL,
    current_value     REAL DEFAULT 0,           -- 현재 평가액 (KRW)
    acquisition_date  TEXT,                     -- YYYY-MM-DD
    acquisition_price REAL DEFAULT 0,           -- 취득가 (단가 또는 총액)
    disposal_date     TEXT,                     -- 매각일 (NULL이면 보유중)
    disposal_price    REAL DEFAULT 0,           -- 매각금액
    quantity          REAL DEFAULT 0,           -- 수량 (주식/실물자산용)
    created_at        TEXT DEFAULT CURRENT_TIMESTAMP,
    updated_at        TEXT DEFAULT CURRENT_TIMESTAMP
);
```

### 3.2 real_estate_details (부동산 상세)
```sql
CREATE TABLE IF NOT EXISTS real_estate_details (
    asset_id      TEXT PRIMARY KEY,
    is_owned      INTEGER DEFAULT 1,    -- 1=자가, 0=임대
    has_tenant    INTEGER DEFAULT 0,    -- 1=세입자 있음
    tenant_deposit REAL DEFAULT 0,     -- 보증금
    address       TEXT,
    loan_amount   REAL DEFAULT 0,       -- 대출금
    FOREIGN KEY (asset_id) REFERENCES assets(id) ON DELETE CASCADE
);
```

### 3.3 stock_details (주식 상세)
```sql
CREATE TABLE IF NOT EXISTS stock_details (
    asset_id           TEXT PRIMARY KEY,
    account_name       TEXT,               -- 계좌명 (증권사)
    currency           TEXT DEFAULT 'KRW', -- KRW|USD|JPY
    is_pension_like    INTEGER DEFAULT 0,  -- 연금식 수령 여부
    pension_start_year INTEGER,
    pension_monthly    REAL,
    ticker             TEXT,               -- Yahoo Finance ticker
    FOREIGN KEY (asset_id) REFERENCES assets(id) ON DELETE CASCADE
);
```

### 3.4 pension_details (연금 상세)
```sql
CREATE TABLE IF NOT EXISTS pension_details (
    asset_id                TEXT PRIMARY KEY,
    pension_type            TEXT,               -- 국민연금|퇴직연금|개인연금 등
    expected_start_year     INTEGER,
    expected_end_year       INTEGER,
    expected_monthly_payout REAL DEFAULT 0,     -- 예상 월 수령액 (KRW)
    annual_growth_rate      REAL DEFAULT 0,     -- 연 증가율 (%)
    FOREIGN KEY (asset_id) REFERENCES assets(id) ON DELETE CASCADE
);
```

### 3.5 savings_details (예적금 상세)
```sql
CREATE TABLE IF NOT EXISTS savings_details (
    asset_id           TEXT PRIMARY KEY,
    is_pension_like    INTEGER DEFAULT 0,
    pension_start_year INTEGER,
    pension_monthly    REAL,
    FOREIGN KEY (asset_id) REFERENCES assets(id) ON DELETE CASCADE
);
```

### 3.6 asset_history (자산 이력)
```sql
CREATE TABLE IF NOT EXISTS asset_history (
    id       INTEGER PRIMARY KEY AUTOINCREMENT,
    asset_id TEXT NOT NULL,
    date     TEXT NOT NULL,   -- YYYY-MM-DD
    value    REAL,            -- 평가액 (KRW, 환율 적용 후)
    price    REAL,            -- 단가 (주식/실물자산용, 원화 또는 외화)
    quantity REAL,            -- 수량 (주식/실물자산용)
    FOREIGN KEY (asset_id) REFERENCES assets(id) ON DELETE CASCADE
);
CREATE INDEX idx_history_asset_id ON asset_history(asset_id);
CREATE INDEX idx_history_date ON asset_history(date);
```

### 3.7 settings (사용자 설정)
```sql
CREATE TABLE IF NOT EXISTS settings (
    key   TEXT PRIMARY KEY,
    value TEXT
);
-- 주요 key: current_age, retirement_age, ACC_TOTAL_{계좌명}
```

## 4. API 설계 개요

### 기본 규칙
- API prefix: `/api/`
- 요청/응답 형식: JSON
- 에러 응답: `{"detail": "에러 메시지"}`
- 자산 ID: UUID v4

### 4.1 자산 (Assets)
| Method | Path | 설명 |
|--------|------|------|
| GET | `/api/assets` | 전체 자산 조회 (이력 포함, `?type=STOCK` 으로 유형 필터) |
| POST | `/api/assets` | 자산 추가 |
| PUT | `/api/assets/{id}` | 자산 수정 |
| DELETE | `/api/assets/{id}` | 자산 삭제 (이력 포함 CASCADE) |
| GET | `/api/assets/chart` | 차트 집계 데이터 (`?type=&period=&group_by=`) |

### 4.2 이력 (History)
| Method | Path | 설명 |
|--------|------|------|
| GET | `/api/assets/{id}/history` | 자산 이력 조회 |
| POST | `/api/assets/{id}/history` | 이력 추가 |
| PUT | `/api/assets/{id}/history/{date}` | 이력 수정 (수량 전파) |
| DELETE | `/api/assets/{id}/history/{date}` | 이력 삭제 (특정 날짜) |

### 4.3 주식 업데이트
| Method | Path | 설명 |
|--------|------|------|
| POST | `/api/stocks/update` | 전체 주식 시세 업데이트 |

### 4.4 설정
| Method | Path | 설명 |
|--------|------|------|
| GET | `/api/settings` | 설정 조회 |
| PUT | `/api/settings` | 설정 저장 |

### 4.5 헬스체크
| Method | Path | 설명 |
|--------|------|------|
| GET | `/api/health` | 서버 상태 확인 |

## 6. 상태 관리 전략

- **서버 상태**: React Query (TanStack Query)
  - `useAssets()` — 전체 자산 목록 (캐싱 5분)
  - `useAssetsByType(type)` — 유형별 필터링
  - `useHistory(assetId)` — 특정 자산 이력
  - `useSettings()` — 설정값
- **클라이언트 상태**: React `useState` / Context
  - 선택된 계좌 (주식 계좌별 보기)
  - 확장된 자산 ID (상세 뷰)
  - 차트 기간 필터

## 7. 차트 데이터 처리 전략

### 배경
- 이력 데이터를 날짜별로 보간(Forward Fill)하여 연속 Area Chart 표현
- 매각된 자산은 매각일 이후 0으로 처리

### 처리 위치: **백엔드 (FastAPI)**
- 클라이언트 복잡도 감소
- 대용량 이력 처리 시 서버 집계가 유리
- API: `GET /api/assets/chart?type=STOCK&period=1y&group_by=account`
- 응답: `[{date, label, value}]` 형태의 집계 데이터 (`label`은 group_by에 따라 유형명/자산명/계좌명)

### 차트 기간 옵션
| 옵션 | 기간 | 대상 |
|------|------|------|
| 전체 | 전체 기간 | 모든 유형 |
| 최근 10년 | 10년 | 대시보드, 부동산 |
| 최근 3년 | 3년 | 기본값 |
| 최근 1년 | 1년 | 주식 |
| 최근 3개월 | 90일 | 주식 |
| 최근 1개월 | 30일 | 주식 |
