# 📄 1단계 요구사항 정의서 (Software Requirements Specification)

## 1. 프로젝트 개요

- **프로젝트명**: My Asset Manager (개인 자산 관리 시스템)
- **목표**: 개인의 전체 자산(부동산·주식·연금·예적금·실물·기타)을 단일 웹 화면에서 체계적으로 관리하고, 시각화를 통해 자산 현황을 직관적으로 파악할 수 있는 전문가 수준의 개인 금융 대시보드 구축
- **기존 시스템**: Streamlit 기반 단일 파일 앱 → FastAPI + React/TypeScript 기반 풀스택 앱으로 전면 개편
- **핵심 가치**:
  - 세련되고 가시성 높은 전문가 풍 UI/UX
  - 자산 유형별 맞춤형 데이터 관리
  - 주가 자동 업데이트 및 이력 관리
  - 단일 사용자 로컬 앱 (Docker 컨테이너)

---

## 2. 기술 스택 (Tech Stack)

### Frontend
- **React 18** + **TypeScript** + **Vite**
- **Tailwind CSS** — 유틸리티 기반 스타일링
- **shadcn/ui** — 고품질 UI 컴포넌트 (Button, Dialog, Form, Table, Card 등)
- **Recharts** — 금융 차트 (Area, Pie, Bar)
- **React Query (TanStack Query)** — 서버 상태 관리, 캐싱
- **React Router v6** — 클라이언트 사이드 라우팅
- **React Hook Form + Zod** — 폼 관리 및 유효성 검사
- **date-fns** — 날짜 처리

### Backend
- **Python 3.11** + **FastAPI** + **Uvicorn** (async)
- **SQLite3** + **aiosqlite** — 경량 로컬 DB (비동기)
- **SQLAlchemy 2.0** (async) — ORM
- **yfinance** — 주가/환율 자동 업데이트
- **APScheduler** — 주가 자동 업데이트 스케줄링

### Infra
- **Docker** + **docker-compose**
- 포트: `8090` (웹 서비스), `1040` (SSH 접속)
- DB 파일: `data/assets.db`
- 프론트엔드 빌드 결과: `/static` (FastAPI가 정적 서빙)

## 3. 기능 요구사항 (Functional Requirements)

### 3.1 대시보드 (Dashboard)
- **종합 KPI 카드**: 총 자산, 총 부채, 순자산 (총자산 - 부채)
- **자산 비중 파이차트**: 유형별(부동산/주식/연금/예적금/실물/기타) 비중 도넛 차트
- **자산 성장 추이**: 최근 10년 Area 차트 (유형별 누적 스택)
- **매각 자산 제외**: 처분된 자산은 현재 자산 합계에서 제외

### 3.2 자산 유형별 관리 페이지
6가지 자산 유형 각각의 페이지:
- `REAL_ESTATE` — 🏠 부동산
- `STOCK` — 📈 주식
- `PENSION` — 🛡️ 연금
- `SAVINGS` — 💰 예적금/현금
- `PHYSICAL` — 💎 실물자산
- `ETC` — 🎸 기타

**공통 기능 (모든 유형)**:
- 카테고리 KPI 요약 (유형별 총자산, 부채, 순자산)
- 유형별 자산 성장 추이 차트 (기간 필터: 전체/10년/3년/1년/3개월/1개월)
- 자산 목록 (카드 또는 Accordion)
- 자산별 상세 뷰:
  - 기본 정보 (취득일, 취득가, 현재가치, 상태)
  - KPI 카드 (유형별 맞춤)
  - 가치 변동 추이 차트
  - 이력 관리 테이블 (조회/추가/수정/삭제)
  - 속성 수정 폼
  - 자산 삭제

**부동산 전용**:
- 추가 필드: 주소, 대출금, 보증금, 자가/임대 여부, 세입자 여부
- KPI: 현재 시세, 부채 총계(대출+보증금), 순자산(Equity)

**주식 전용**:
- 추가 필드: 계좌명, 통화(KRW/USD/JPY), Ticker(Yahoo Finance)
- 보기 모드: **계좌별 보기** / **종목별 보기** 전환
- 계좌별 보기: 계좌 선택 카드 그리드 → 선택 시 KPI + 차트 + 종목 목록
- **주가 자동 업데이트**: yfinance 기반 일괄 업데이트 버튼
- 이력: 단가 + 수량 기반 (수량 변경 시 이후 이력 전파)

**연금 전용**:
- 추가 필드: 연금 유형, 수령 시작/종료 연도, 예상 월 수령액, 연 증가율(%)
- **노후 월 수령액 시뮬레이션**: 향후 60년 스택 Bar 차트

**예적금 전용**:
- 연금 유사 옵션 (만기 시 연금 전환)

### 3.3 이력 관리 (Asset History)
- 날짜별 자산 가치 이력 기록
- 주식/실물자산: 단가 + 수량 + (자동계산) 평가액
- 그 외: 날짜별 평가액
- 이력 수정: 특정 날짜 선택 후 단가/수량/평가액 수정
- 수량 변경 시 이후 날짜 이력에 수량 자동 전파 (단가는 유지)
- 이력 삭제: 특정 날짜 단건 삭제

### 3.4 주가 자동 업데이트 (Stock Auto-Update)
- yfinance를 통해 Ticker가 설정된 모든 주식 종목의 시세 자동 조회
- 마지막 이력 날짜부터 현재까지 Backfill
- 환율 자동 적용 (USD/JPY → KRW 변환)
- API 엔드포인트 또는 APScheduler를 통한 정기 실행

### 3.5 설정 (Settings)
- 현재 나이 (연금 시뮬레이션용)
- 은퇴 목표 나이

## 4. 비기능 요구사항 (Non-Functional Requirements)

### 4.1 UI/UX
- 다크모드/라이트모드 지원 (shadcn/ui 기본 테마)
- 반응형 레이아웃 (데스크톱 우선)
- 한국어 UI

### 4.2 성능
- 자산 목록 로딩: 500ms 이하
- 차트 데이터 생성: 클라이언트 or 서버 사이드 집계
- React Query를 통한 캐싱 (staleTime 5분)

### 4.3 데이터 무결성
- SQLite FK 강제 (ON DELETE CASCADE)
- 매각일 이후 이력 추가 불가 (유효성 검사)
- DB 파일은 `data/` 디렉토리에 영속적으로 보존

### 4.4 운영
- Docker 기반 단일 컨테이너 배포
- 프론트엔드 빌드 후 `/static`에 서빙 (별도 Nginx 불필요)
- 로그: `logs/server.log`

## 5. 데이터 모델 개요 (Database Schema Summary)

| 테이블 | 설명 |
|--------|------|
| `assets` | 모든 자산 공통 정보 |
| `real_estate_details` | 부동산 상세 (주소, 대출, 보증금 등) |
| `stock_details` | 주식 상세 (계좌명, 통화, Ticker 등) |
| `pension_details` | 연금 상세 (수령 시기, 월 수령액 등) |
| `savings_details` | 예적금 상세 (연금 전환 옵션) |
| `asset_history` | 날짜별 자산 가치 이력 |
| `settings` | 사용자 설정 (나이, 은퇴 목표 등) |
